# AI-Product-OPS

Репозиторий AI-агентов для продуктового офиса. Каждый агент решает конкретную задачу аналитика или продакт-менеджера и доступен как самостоятельное приложение.

---

## Агенты

### 1. [Classificator](./Classificator) — Классификатор инициатив

**Что делает:** Определяет, является ли инициатива **Продуктом** или **Проектом**, и выдаёт обоснованное заключение с процентной оценкой.

**Зачем это нужно:** Продукт и проект требуют разных моделей управления. Ошибка в классификации приводит к неверному выбору команды, процессов и критериев успеха. Агент помогает сделать этот выбор осознанно.

**Как работает:**
- Принимает материалы инициативы (презентация, описание, шаблон) или ведёт диалог с нуля
- Задаёт уточняющие открытые вопросы по 8 ключевым критериям: стратегический фокус, измерение успеха, источник требований, формирование бэклога, приоритезация, процесс, модель управления, горизонт
- Отслеживает уровень неопределённости и не делает вывод раньше времени
- Выдаёт двухшаговое заключение: сначала сводка фактов на подтверждение, затем скоринг + классификация + рекомендации

**Вывод:** скоринг в % · классификация (Продукт / Предварительно продукт / Неопределённо / Проект) · рекомендации по усилению модели управления

**Стек:** Python · Anthropic SDK (Claude) · Streamlit

```bash
cd Classificator
pip install -r requirements.txt
streamlit run app.py
```

### 2. [Vision Creator](./Vision/VisionCreator) — Формирование продуктового видения

**Что делает:** Через итеративный диалог помогает Product Owner сформулировать структурированное продуктовое видение — компактное, с фокусом на аудиторию, ценность и стратегическое направление.

**Зачем это нужно:** Размытое или отсутствующее видение — главная причина, по которой команды работают без ориентира. Агент задаёт правильные вопросы, отличает цели от метрик, стратегию от roadmap, и собирает результат в готовый документ.

**Как работает:**
- Принимает материалы (описание, презентации, метрики) или работает с нуля, а также дорабатывает существующее видение
- Отслеживает неопределённость по 12 разделам и не предлагает генерировать результат раньше времени
- Проверяет здравый смысл: если идея нереалистична или противоречива — мягко уточняет
- Генерирует видение только при неопределённости ≤ 0.01 (или ≤ 0.05 по запросу PO)

**Вывод:** формулировка видения · стратегия по годам с ориентирами · бизнес-цели с цифрами · контекст и риски

**Стек:** Python · Anthropic SDK (Claude) · Streamlit

```bash
cd Vision/VisionCreator
pip install -r requirements.txt
streamlit run app.py
```

### 3. [Vision Assessor](./Vision/VisionAssessor) — Оценка продуктового видения

**Что делает:** Принимает готовый документ продуктового видения и выдаёт структурированный отчёт по 6 критериям с оценкой ИЗИ и итоговым вердиктом.

**Зачем это нужно:** Быстро проверить видение перед тем, как нести его в команду или стейкхолдерам. Агент оценивает строго по явному содержанию — без додумываний, с цитатами из текста.

**Как работает:**
- Принимает текст видения целиком (формулировка + стратегия + цели + контекст)
- Оценивает каждый из 6 критериев: Ok / Частично / Не проходит
- Считает оценку ИЗИ (0-2) и итоговый счёт (N из 6)
- Выдаёт вердикт: «Готово к использованию» / «Нужна доработка» / «Требует фундаментальной переработки»

**Критерии:** структурная целостность · временной горизонт (3+ лет) · клиентоцентричность · ценность для бизнеса · цели и метрики · связь со стратегией

**Стек:** Python · Anthropic SDK (Claude) · Streamlit

```bash
cd Vision/VisionAssessor
pip install -r requirements.txt
streamlit run app.py
```

### 4. [Hypothesis Generator](./Discovery/Hyp.Generator) — Генератор продуктовых гипотез

**Что делает:** Собирает контекст о продукте из материалов или диалога и генерирует набор продуктовых гипотез в строгом формате с уровнем уверенности, обоснованием и приоритетным рейтингом.

**Зачем это нужно:** Гипотезы без данных — это догадки. Агент не придумывает цифры: каждое утверждение имеет источник, а где данных нет — явно ставит метку. Помогает команде понять, что тестировать в первую очередь и что нужно собрать.

**Как работает:**
- Принимает любые материалы: видение, брифы, метрики, исследования, бэклог
- Заполняет 6 блоков контекста и не генерирует до заполнения минимум 2 блоков
- Формирует 5-10 гипотез в формате «Если мы [действие], то [метрика]...» с указанием источника каждого факта
- Ранжирует по частоте сигнала, уровню уверенности и близости к бизнес-метрикам

**Вывод:** гипотезы с уверенностью · рейтинг · пробелы в данных · рекомендации по тестированию

**Стек:** Python · Anthropic SDK (Claude) · Streamlit

```bash
cd Discovery/Hyp.Generator
pip install -r requirements.txt
streamlit run app.py
```

### 5. [Brief Writer](./Discovery/Brief) — Написание продуктового брифа

**Что делает:** Через итеративный диалог собирает всю необходимую информацию об инициативе и формирует полный структурированный бриф, готовый к передаче в команду разработки.

**Зачем это нужно:** Без чёткого брифа команда строит не то. Агент задаёт правильные вопросы бизнес-аналитика, следит за полнотой информации и не генерирует документ, пока не закрыты все критические разделы.

**Как работает:**
- Принимает описание инициативы в любом формате или начинает диалог с нуля
- Отслеживает неопределённость по формуле: 7 критических разделов (вес 70%) + 5 вспомогательных (вес 30%)
- Не формирует бриф до неопределённости ≤ 10%
- Показывает сводку фактов на подтверждение перед финальным документом

**Вывод:** бриф из 12 разделов: скоуп · Job Story · критерии приёмки · RICE · метрики · риски

**Стек:** Python · Anthropic SDK (Claude) · Streamlit

```bash
cd Discovery/Brief
pip install -r requirements.txt
streamlit run app.py
```
